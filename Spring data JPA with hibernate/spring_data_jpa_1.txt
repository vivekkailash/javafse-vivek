Country.java   (com.cognizant.ormlearn.model)
---------------------------------------------
package com.cognizant.ormlearn.model;

import javax.persistence.Column;
import javax.persistence.Entity;
import javax.persistence.Id;
import javax.persistence.Table;

@Entity
@Table(name="country")
public class Country {

    @Id
    @Column(name="co_code")
    private String code;

    @Column(name="co_name")
    private String name;

    // getters + setters
    public String getCode() { return code; }
    public void setCode(String code) { this.code = code; }

    public String getName() { return name; }
    public void setName(String name) { this.name = name; }

    @Override
    public String toString() {
        return "Country [code=" + code + ", name=" + name + "]";
    }
}

package com.cognizant.ormlearn.service;

import java.util.List;

import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

import com.cognizant.ormlearn.model.Country;
import com.cognizant.ormlearn.repository.CountryRepository;

@Service
public class CountryService {

    @Autowired
    private CountryRepository countryRepository;

    @Transactional
    public List<Country> getAllCountries() {
        return countryRepository.findAll();
    }
}

Hands on 4 - Difference between JPA, Hibernate & Spring Data JPA
----------------------------------------------------------------

** notes (no code needed) **

JPA (Java Persistence API)
- JSR 338 specification
- just a specification - no implementation
- Hibernate is one implementation of JPA

Hibernate
- ORM tool that implements JPA
- session factory, session, transaction etc

Spring Data JPA
- another layer above JPA / Hibernate
- reduces boilerplate code
- manages transactions
- gives us JpaRepository with save, findById, findAll etc


Hibernate style code sample (long):
   Session session = factory.openSession();
   Transaction tx = null;
   try {
       tx = session.beginTransaction();
       employeeID = (Integer) session.save(employee);
       tx.commit();
   } catch (HibernateException e) {
       if (tx != null) tx.rollback();
   } finally {
       session.close();
   }


Spring Data JPA style (much shorter):
   @Autowired
   private EmployeeRepository employeeRepository;

   @Transactional
   public void addEmployee(Employee employee) {
       employeeRepository.save(employee);
   }

** big difference - spring data JPA hides all the session/tx code **

-------------------------------------------------------

** run the big insert script that has all ~250 countries **
** file given: country insert list **

Also change ddl-auto:
# in application.properties
spring.jpa.hibernate.ddl-auto=validate     # keep as validate

** validate = just check tables exist. safest for prod-like work **

Notes on ddl-auto:
- create      -> drops and recreates tables (dangerous - loses data)
- validate    -> just checks table + columns exist (throws if missing)
- update      -> adds missing tables/columns, doesn't drop
- create-drop -> creates on start, drops on shutdown

CountryService.java  (add this method)
--------------------------------------
@Transactional
public Country findCountryByCode(String countryCode) throws CountryNotFoundException {

    Optional<Country> result = countryRepository.findById(countryCode);

    if (!result.isPresent()) {
        throw new CountryNotFoundException("Country not found for code: " + countryCode);
    }
    return result.get();
}

// don't forget: import java.util.Optional;

public void updateCountry(String code, String name) {
    Country country = countryRepository.findById(code).get();
    country.setName(name);
    countryRepository.save(country);
}
